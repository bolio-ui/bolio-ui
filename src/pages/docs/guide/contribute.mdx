## Contributing to Bolio UI

<Spacer h={2} />

### Ways to contribute

First and foremost, we appreciate your interest in contributing to Bolio UI! Your contributions play a crucial role in enhancing the project.
Here are various ways you can get involved

- • <Text b>Enhance documentation:</Text> Address incomplete or missing documentation,
  improve wording, examples, or explanations.

- • <Text b>Provide feedback:</Text> We are continually striving to improve Bolio UI. Please share your usage experiences, highlight missing features, and let us know what works well through <a href="https://github.com/bolio-ui/bolio-ui/discussions" rel="noreferrer" target="_blank">GitHub Discussions</a>.

- • <Text b>Promote Bolio UI:</Text> Share links to the Bolio UI documentation with anyone who might find it useful.

- • <Text b>Contribute to the codebase:</Text> Propose new features by opening <a href="https://github.com/bolio-ui/bolio-ui/issues/new" rel="noreferrer" target="_blank">GitHub Issues</a>, or take on <a href="https://github.com/bolio-ui/bolio-ui/labels/" rel="noreferrer" target="_blank">existing issues</a> that align with your interests.

- • <Text b>Conduct a code review:</Text> Help us identify issues in the <a href="https://github.com/bolio-ui/bolio-ui/tree/master/src" rel="noreferrer" target="_blank">source code</a> or suggest improvements to make Bolio UI more efficient.

<Spacer h={2} />

### How to contribute?

- • Decide on the contribution you want to make.

- • For new features, discuss your ideas with the maintainer (via <a href="https://github.com/bolio-ui/bolio-ui/discussions" rel="noreferrer" target="_blank">GitHub Discussions</a>) before starting the coding process.

- • Once issue details are finalized, adhere to commit conventions as you work on the code.

- • Run tests using `npm test` and submit a pull request (pull request) after ensuring all tests pass.

- • Undergo a code review, addressing any issues identified by the maintainer.

- • If you're unable to complete the task or have a change of plans, no worries! Just inform us in the GitHub issue you initially created.
  The Bolio UI community is understanding – no judgment or questions if you decide to cancel your submission.

- • Your pull request gets merged!"
